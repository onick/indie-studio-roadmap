# Estrategia de Producto B2B SaaS

## Vision en 1-2 Frases

Una buena vision responde: para quien, que problema, que resultado.

Mal: "Un PMS moderno y completo."
Bien: "Hoteles independientes del Caribe gestionan reservas, housekeeping y facturacion desde una sola plataforma, sin depender de sistemas legacy de $500/mes."

## Jobs to Be Done (JTBD)

No vendas software — vende el resultado que el cliente necesita.

Preguntas clave:
- Cuando [situacion], quiero [accion], para [resultado esperado].
- Ejemplo: "Cuando llega un huesped sin reserva, quiero registrarlo en menos de 2 minutos, para no hacer esperar a la fila."

Cada modulo del producto debe mapear a un JTBD concreto.

## Fases que Generan Valor Incremental

El error mas comun: construir 100% de todo antes de lanzar. En B2B, cada fase debe dejar al cliente con algo usable.

Framework de priorizacion:
1. **Fase Core:** Lo minimo para que un hotel opere un dia normal (reservas + check-in + checkout + factura basica).
2. **Fase Operativa:** Lo que mejora operaciones diarias (housekeeping, reportes, inventario).
3. **Fase Comercial:** Lo que genera mas ingresos (channel manager, revenue management, CRM).
4. **Fase Escala:** Lo que permite crecer (multi-propiedad, API publica, marketplace).

## Modulo Minimo Desplegable (MMD)

El fragmento mas chico de funcionalidad que un hotel puede usar y obtener valor. No es un endpoint — es un flujo completo:

- MMD de Reservas: crear reserva + ver lista + cancelar. No necesita drag-and-drop calendar para ser util.
- MMD de Housekeeping: ver estado de habitaciones + marcar limpia. No necesita asignacion de personal para empezar.

## Pricing para SaaS Hotelero

Modelos comunes:
- **Per-property:** Precio fijo por hotel ($49-199/mes). Simple, predecible.
- **Per-room:** Precio por habitacion ($1-5/hab/mes). Escala con el cliente.
- **Tiered:** Free (10 hab) → Pro (50 hab) → Enterprise (ilimitado). Mejor para acquisition.

Regla: No compliques el pricing al inicio. Empieza con per-property, 1 tier, y ajusta cuando tengas 10+ clientes.

## Posicionamiento Indie vs Enterprise

No compitas en features contra Opera/Cloudbeds. Compite en:
- **Velocidad de setup:** "En 15 minutos estas operando" vs semanas de implementacion.
- **Precio:** 10x mas barato que enterprise.
- **Soporte personal:** Hablas con el fundador, no con un ticket.
- **Localismo:** Entiendes la realidad del Caribe (RNC, ITBIS, temporadas, Booking.com).
